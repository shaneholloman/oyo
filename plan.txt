Refactor plan: deferred diff + background compute for large files

Goal
- Avoid UI hangs on huge text while still allowing stepping once diff completes.
- Diffing large files is deferred, computed in background, and upgrades in place.

Plan
1) Core diff state + upgrade hooks
   - Add DiffStatus (Ready/Deferred/Computing/Failed/Disabled) in oyo-core multi.
   - Replace precomputed_diffs with a slot enum (Placeholder/Ready) + diff_statuses.
   - Preserve original old/new contents when deferring; placeholder diff uses display-only text.
   - Expose diff_status accessors and setters (mark computing/failed, apply diff result).
   - When diff result arrives, reset navigator, update insertions/deletions.

2) Diff worker + scheduling (app)
   - Add DiffRequest/DiffResponse types (app/types.rs).
   - Add diff worker channels + queue to App; spawn worker on demand.
   - Queue diff for current file on enter if status == Deferred.
   - Idle scheduling: after N ms without input and no inflight job, queue next deferred file.
   - Poll responses in tick(); apply diff result and refresh caches.
   - Preserve scroll + cursor; avoid stepping until diff is Ready.

3) Config + docs + tests
   - ui.diff.max_bytes becomes the defer threshold (large files are deferred, not disabled).
   - Add ui.diff.defer (default true) + ui.diff.idle_ms (default 250) if needed.
   - Update README + DIFF_VIEWER docs to explain deferred diff and behavior.
   - Tests: deferral state, worker upgrade (status Ready + navigator reset), counts update.
